# 订单表格底部汇总行

## 目标

将全部订单页面原有勾选统计卡片替换为方案 B：表格底部“已选合计”汇总行。

## 范围

- 明细表和汇总表各自使用 `el-table` 汇总行展示已选统计。
- 父级全部订单页面继续管理两种表格的跨分页勾选状态。
- 明细表统计：照片总张数、接单总价、派单总价。
- 汇总表统计：订单总数、照片总张数、接单总价、派单总价、利润。
- 汇总行作为表格内部 footer，由表格组件自然扣除高度，不遮挡数据行。

## 验收

- 勾选明细表数据后，表格底部显示明细“已选合计”汇总行。
- 勾选汇总表数据后，表格底部显示汇总“已选合计”汇总行。
- 未勾选时表格使用完整高度。
- `npm run build` 通过。
